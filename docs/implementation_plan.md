# 관리자 페이지 고도화 및 이스터에그 구현 계획

사용자가 더 쉽고 직관적으로 관리자 기능에 접근하고 사용할 수 있도록 개선합니다. 또한 데이터 무결성을 보장하기 위해 저장 로직의 버그를 수정합니다.

## 주요 개선 사항

### 1. 관리자 페이지 접근성 개선 (이스터에그)
- **접속 방식**: 메인 페이지(`index.html`)의 프로필 사진을 **5번 연속 클릭**하면 자동으로 `admin.html`로 이동합니다.
- **사용자 경험**: 별도로 URL을 입력할 필요 없이 숨겨진 기능을 찾듯 자연스러운 접근이 가능합니다.

### 2. 관리자 페이지 UI 개편
- **제목 변경**: 불필요한 수식어를 제거하고 깔끔한 제목으로 수정합니다.
- **탭 세분화**: 논문 실적 탭을 **Journal / Conference / Patent** 3개의 하위 탭으로 분리하여 많은 양의 데이터를 관리하기 쉽게 개선합니다.
- **레이아웃 수정**: 제목이나 텍스트가 잘리는 현상을 스타일 수정을 통해 해결합니다.

### 3. 데이터 저장 로직 수정 (버그 픽스)
- **편집 반영**: 기존 내용의 글자를 수정해도 반영되지 않던 문제를 `updateItem` 로직 개선을 통해 해결합니다.
- **특허 누락 해결**: 특허(Patent) 데이터가 빈 배열(`[]`)로 저장되던 데이터 분류 로직 버그를 수정합니다.

---

## 작업 단계

1.  **`index.html` 수정**: 프로필 사진 클릭 카운터 및 이동 로직 추가.
2.  **`admin.html` 수정**: 
    - 하위 탭 UI 구현 및 스타일 수정.
    - 데이터 로드 시 `category` 필드 강제 할당 (분류 버그 해결).
    - `updateItem` 함수가 실제 데이터 상태를 즉각 반영하도록 수정.
3.  **검증**: 모든 탭에서 데이터 추가/수정/삭제 후 다운로드하여 JSON 파일 내용이 완벽한지 확인.

## 검증 계획

- **이스터에그 테스트**: 사진 5회 클릭 시 페이지 이동 확인.
- **데이터 무결성 테스트**: 특허 항목을 추가/수정한 후 다운로드한 JSON이 유효한지 확인.
- **반응형 테스트**: 제목이나 폼 요소가 모바일/데스크탑에서 잘리지 않는지 확인.
